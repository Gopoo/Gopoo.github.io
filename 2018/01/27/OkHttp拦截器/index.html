OkHttp拦截器

发表于 2018-01-27

OkHttp中的拦截器

  • okhttp提供了拦截器链进行网络请求及回传数据解析，包含：RetryAndFollowUpInterceptor、BridgeInterceptor、CacheInterceptor、ConnectInterceptor、CallServerInterceptor
  • 无论是同步的execute()还是异步的enqueue()都会调用getResponseWithInterceptorChain（）获取responce，并放入callback中

创建拦截器集合、拦截器链对象（将interceptors，index ：0等传给构造函数），并调用chain.proceed()

RealInterceptorChain

  • A concrete interceptor chain that carries the entire interceptor chain: all application interceptors, the OkHttp core, all network interceptors, and finally the network caller.

proceed方法中创建index为当前index+1的拦截器链对象，获取当前index对应的拦截器对象，调用intercept方法。在该方法中，调用index+1的proceed()进行创建index为index+1+1的拦截器链对象，获取index+1的拦截器，调用intercept方法……一直循环创建直到index>= interceptors.size()拦截器链创建并执行完成

RetryAndFollowUpInterceptor

  • This interceptor recovers from failures and follows redirects as necessary.
  • 重试重定向拦截器：请求失败后重试，请求时重定向

拦截器intercept()中获取拦截器链对象，创建StreamAllocation对象，在死循环中调用拦截器链的proceed，执行次数大于20不再请求。

BridgeInterceptor

拦截器添加一些请求头部信息，keep-alive、gzip压缩，调用拦截器链proceed()，对返回的数据进行gzip解析

CacheInterceptor

CacheInterceptor使用DiskLRUCache对GET请求进行缓存，如果返回304，使用缓存

ConnectInterceptor

使用RetryAndFollowUpInterceptor中创建的streamAllocation的newStream()方法

findHealthyConnection中一直循环调用findConnection，直到Connection是isHealthy或新建的，在findConnection中进行TCP、TSL握手

CallServerInterceptor

写入request，获取responce并返回
